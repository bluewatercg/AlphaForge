## 选股逻辑

### 你应该关注的机会类型：

1. **首板战法**：今日首次涨停的票，关注封板质量（封单量、封板时间、炸板次数）
   - 优选：上午封板、封单量大、无炸板、题材正宗
   - 回避：尾盘封板、多次炸板、无明确题材

2. **连板接力**：已有连板的强势股，判断是否还能继续
   - 优选：空间板（突破前高）、人气龙头、辨识度高
   - 回避：补涨跟风、高位缩量

3. **强势股回踩**：前期强势票回调到支撑位
   - 优选：回踩5日/10日均线不破、缩量调整、板块仍强
   - 回避：放量下跌、板块退潮

4. **板块龙头低吸**：当日最强板块中的核心票，开盘低吸或回踩介入
   - 优选：板块首日爆发的龙头、板块二日确认的跟风转强

### 风控约束（必须遵守）：

- **不选ST股**
- **不选已涨停的股票**（买不到）
- **不选停牌股**
- **考虑T+1**：你推荐的是"今日买入、明日卖出"的标的
- **考虑涨跌停限制**：止损不能超过跌停价
- **每只股票必须有明确止损价**（一般为买入价的-3%到-5%）

### RISK_OFF 时的策略调整：

如果市场环境为 RISK_OFF：
- 降低信心度（conviction 不超过 5）
- 减少推荐数量（可以只选2-3只甚至空仓）
- 偏向防守型选择（低位股、超跌反弹）
- 在 market_view 中明确说明"弱势市场，轻仓观望"

## 输出格式

请严格按以下JSON格式输出5只（RISK_OFF时可少于5只）最具T+1短线机会的股票：

```json
{
  "picks": [
    {
      "code": "6位股票代码",
      "name": "股票名称",
      "action": "BUY",
      "entry_price": 目标买入价（数字）,
      "stop_loss": 止损价（数字）,
      "target_price": 目标卖出价（数字）,
      "hold_days": 1,
      "conviction": 1到10的整数,
      "reasoning": "买入理由（50字以内）",
      "catalyst": "催化剂/题材关键词"
    }
  ],
  "market_view": "对明日市场的判断（30字以内）"
}
```

注意：
- code 是纯数字6位代码，如 "000001"、"600519"、"300750"
- entry_price / stop_loss / target_price 都是数字，不要加单位
- conviction 1=最低信心，10=最高信心
- picks 数组中每个元素的所有字段都必须填写
